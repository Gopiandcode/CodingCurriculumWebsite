    Now, this system doesn’t seem to do much – in order to provide functionality, we need to add a way of storing information about the patient’s symptoms and some way to store the cure that we calculate.
    Add a string array to each patient to store their symptoms.

  • 3. It looks like there's a bug on the system - patient data is being changed.

    It looks like there’s a bug somewhere else in the system which messes with the patient data.
    While one solution could be to fix the bug itself, but when working with systems as big as Watson, one developer only designs a small section of code.
    Thus, it’s our responsibility to make sure that, irrespective of what other people might try to do to our code, we make it work.
    Try implementing encapsulation in this system to make it work. (Hint: try adding private)

  • 4. Implement methods to selectively allow use of the private details.

    Now the class is encapsulated, our code is protected, however now, even Watson can’t access it.
    We need some way of allowing people to view or modify only certain sections of our code. This can be done by implementing a getter and setter. . Consider patient symptoms – Watson needs access to it, so we can allow Watson access by implementing a getSymptoms function, which returns a copy of the application symptoms.
    Try adding getters and setters for the other fields.
